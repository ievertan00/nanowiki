Loop 01 · Ask Question → Note

你怎么用

先问一个
真实问题

$ /wiki-ask "我该怎么理解 RAG 里的重排序?"
pass 1 free answer
pass 2 refine until useful
save notes/rag-reranking.md

Input

你追问、修正、补上下文

先理解，不急着入库。

Output

带结构的永久笔记

事实、综合、连接、开放问题。

Result · 一条可引用、可链接、可继续生长的 Markdown 笔记

Loop 02 · Ingest Source → Graph

你怎么用

把资料
喂进去

Source

论文 · 文章 · URL · 草稿

原始资料进入 sources/，作为以后可追溯的证据。

Command

/wiki-ingest paper.pdf

先写文献笔记，再找旧笔记里能更新的地方。

01 · literature生成一篇文献笔记
02 · fan out更新相关旧笔记
03 · ledger记录 ingested.json
04 · index重建 MOC 与索引

Result · 资料不是被收藏，而是进入你的知识网络

Loop 03 · Lint Vault → Worklist

你怎么用

定期给
知识库体检

$ /wiki-lint
scan contradictions
scan orphan notes
scan thin notes
write meta/lint-2026-06-25.md

01

找出互相冲突的说法

02

发现孤立笔记

03

列出下一批该问的问题

04

清理分类和缺链

Result · 它不假装全自动正确，而是把该判断的地方交还给你

What You Get Plain Files

最后得到什么

不是聊天记录
是工作资产

01

notes/ 原子笔记每个问题都落成一页，可搜索、可链接、可改写。

02

sources/ 原始资料文章、PDF、草稿保留原貌，不被模型覆盖。

03

moc/ 内容地图按 domain 自动生成，Obsidian 里直接浏览。

04

meta/ 维护报告索引、日志、lint 报告、待写问题持续更新。

Files you own · Markdown · Git-friendly · Obsidian-compatible

Why Useful Real Work

为什么对生活和工作有用

它帮你
把认知留住

Research
读完资料后，不只存链接/wiki-ingest 把重点写成文献笔记，并更新相关旧主题。
Decision
做方案时，先问已有知识库/wiki-query 只从你的笔记回答，减少凭印象乱想。
Writing
写文章时，材料已经连好旧问题、旧来源、旧洞察能被重新组合。
Review
每周复盘，不靠手动翻文件/wiki-lint 把矛盾、薄弱页、下一步问题列出来。

Advanced Loop · Templates Persona + Structure

更像真实工作的一层

给 AI 一个
思考姿态

$ wiki ask "Should we enter this market?" \
  -p investor-decisionmaker \
  -s five-forces

$ wiki ingest paper.pdf \
  -p research-reviewer \
  -s ml-paper-notes

Persona · -p

决定它怎么想

投资人、研究审稿人、费曼解释者、反向思考。

Structure · -s

决定它别漏什么

五力模型、ML 论文笔记、SWOT、技术深挖。

templates/personas你可以新增自己的判断框架
templates/structures你可以固定自己的检查清单

Result · 同一个问题，可以按不同职业场景产出不同质量的笔记
