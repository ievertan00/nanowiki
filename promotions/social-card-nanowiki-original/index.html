Case B · Paper Reading03 / 09

案例 B：读一篇论文

论文不是
存进收藏夹

$ wiki ingest attention-paper.pdf \
  -p research-reviewer \
  -s ml-paper-notes
01
生成 literature note核心观点、方法、实验、局限，都落到一个源头笔记里。
02
反哺已有 atomic notes和旧主题相关的新发现，会追加到对应笔记，不另造一堆碎片。
03
自动重建 MOC / index / log你不需要手动维护目录和索引。
source first · graph secondno manual filing

Files You Own08 / 09

一轮之后真正得到什么

不是漂亮回答
是可维护资产

wiki-vault/
├─ sources/ 原始回答、文章、PDF、URL
├─ notes/ 原子笔记与文献笔记
├─ moc/ 自动生成内容地图
├─ meta/ index、log、lint、questions
└─ templates/ 你自己的 persona / structure
关键区别：文件是你的，不是某个 App 里的黑盒记忆。可读、可改、可搜索、可用 Git 管。
plain markdownobsidian compatible
