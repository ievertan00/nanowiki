## 正文

很多 AI 笔记工具的问题是：  
它们把“聊天记录”保存下来了，但没有真正变成你以后能复用的知识。

我做 Nanowiki 时，想解决的不是“怎么更快记笔记”，而是：

**一个真实任务结束后，我到底得到了什么？**

这版图用几个实际案例演示它的用法。

**案例 1：做市场判断**  
不是直接问 AI“要不要做”，而是给它一个工作框架：

```bash
wiki ask "Should we enter the AI note-taking market?" \
  -p investor-decisionmaker \
  -s five-forces
```

`persona` 决定它用什么视角思考，`structure` 决定它不要漏掉哪些维度。  
结果是一页可以持续更新的市场判断笔记，而不是一次性回答。

**案例 2：读一篇 ML 论文**

```bash
wiki ingest attention-paper.pdf \
  -p research-reviewer \
  -s ml-paper-notes
```

它会生成一篇 literature note，同时把论文里相关的新信息追加到已有主题里。  
论文不是进收藏夹，而是进入你的知识图谱。

**案例 3：学一个技术概念**

```bash
wiki ask "What is attention?" -p feynman-explainer
```

先用容易理解的方式解释，再保存成标准 Wiki note。  
每条笔记都有固定结构：Source Facts、Synthesis、Connections、Open Questions、Human Insight。

**案例 4：写方案前查自己的库**

```bash
wiki query "What do my notes say about RAG evaluation?"
```

`query` 是闭卷的：只从你的已有笔记里回答，并带 `[[note]]` 引用。  
适合写方案、做复盘、查“我以前到底怎么想过这个问题”。

**案例 5：周末给知识库体检**

```bash
wiki lint
```

它会找矛盾、孤立笔记、薄弱笔记和下一批该问的问题。  
这一步很关键：Nanowiki 不假装 AI 永远正确，而是把需要你判断的地方暴露出来。

最后你得到的不是一个 App 里的黑盒数据，而是一组本地文件：

- `sources/`：原始回答、文章、PDF、URL
- `notes/`：原子笔记与文献笔记
- `moc/`：自动生成的内容地图
- `meta/`：index、log、lint 报告、wanted notes、questions
- `templates/`：你自己的 persona 与 structure

适合用在项目研究、论文阅读、技术学习、写作、复盘、长期主题追踪。

开源项目：github.com/ievertan00/nanowiki
